# Architecture: The Deterministic AI Agency

This monorepo is designed to solve the "Generic AI Aesthetic" problem. By utilizing Zod schemas, Abstract Syntax Tree (AST) parsing, and strict Prompt Chaining, we force Large Language Models (OpenAI, Claude, Gemini) to act as a disciplined digital agency rather than a stochastic text generator.

## 🧠 The Agentic Pipeline (Prompt Chaining)

The system is broken into three sequential agents. An agent cannot begin its work until the previous agent's strict JSON output is validated.

### 1. The Orchestrator (UX & Brand Strategy)
* **Role:** Interacts with the human client. It refuses to write code or pick colors. 
* **Mechanism:** It conducts a discovery interview. Once it understands the business goals, target audience, and emotional resonance, it outputs a `StrategyBrief.json`.
* **Guardrail:** Validated via Zod. It must contain specific psychological and demographic data points.

### 2. The Architect (Design Systems Engineering)
* **Role:** Translates the human strategy into hard mathematics.
* **Mechanism:** Reads `StrategyBrief.json` and generates `BrandBlueprint.json`. 
* **Guardrail:** The Zod schema mathematically forces the LLM to output a Major Third (1.250) typography scale and guarantees that every background color has a paired foreground color meeting WCAG AA 4.5:1 contrast ratios.
* **Compilation:** A Node script (`compile.ts`) immediately translates this JSON into global CSS variables.

### 3. The Builder (Frontend Production)
* **Role:** Writes the final interactive React code.
* **Mechanism:** Reads the local codebase AST (via `tools/ai-context-builder`) to understand the exact props of our internal `@craft/ui-system`. 
* **Guardrail:** The LLM is strictly sandboxed. It cannot write raw HTML (`<button>`) or inline CSS. It can only compose the highly-polished, pre-styled components generated by The Architect.

## 🏗 Directory Flow
1. `apps/showcase-web/` -> Hosts the Multi-Agent Chat Interface.
2. `packages/design-architect/` -> Houses the Zod schemas and Agent 1/2 prompts.
3. `packages/theme-core/` -> Compiles the Architect's JSON into CSS.
4. `packages/ui-system/` -> The headless React components that inherit the CSS.
5. `packages/gen-ui/` -> The Agent 3 live-rendering engine.
